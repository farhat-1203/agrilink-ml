# 🌿 AI Crop Quality Estimator

**Single-photo crop quality analysis** — disease detection (MobileNetV3 / EfficientNet-B0)
+ image-based freshness estimation → Grade A/B/C with shelf-life and urgency prediction.

---

## Architecture

```
Single input image  (bytes / base64 / numpy / PIL)
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│  InferenceEngine.predict()                                   │
│                                                             │
│  Step 1 — Decode                                (~1 ms)     │
│    bytes / base64 / numpy / PIL → PIL RGB + numpy array     │
│                                                             │
│  Step 2 — Disease Classifier  (MobileNetV3)    (~55 ms CPU) │
│    Backbone (ImageNet pretrained, partially frozen)         │
│    → Shared projection (512-dim embedding)                  │
│    → disease_head: softmax over N disease classes           │
│    → binary_head : sigmoid P(image is diseased)             │
│                                                             │
│  Step 3 — Freshness Estimator (OpenCV heuristics) (~5 ms)  │
│    6 physics-based signals from HSV / LAB / Laplacian:      │
│    saturation · greenness · browning · sharpness ·          │
│    brightness · uniformity                                  │
│    Weighted sum → freshness_score 0–100                     │
│    Disease penalty: − disease_prob × 25 pts                 │
│                                                             │
│  Step 4 — Decision Engine     (pure Python)      (<1 ms)   │
│    Grade A/B/C  +  shelf_life_days  +  urgency_level        │
│    +  market_tier  +  human-readable summary                │
└─────────────────────────────────────────────────────────────┘
        │
        ▼
QualityReport dict  →  FastAPI response  /  Streamlit display
```

### Why this design?

| Choice | Reason |
|--------|--------|
| MobileNetV3-Small | 2.5M params, ~55ms/image on CPU — deployable on edge devices |
| Dual output heads | Disease class (main) + binary healthy/diseased (freshness signal) |
| Heuristic freshness | PlantVillage has no freshness labels; physics-based CV is interpretable |
| Heuristic fallback | API serves requests before training completes |
| Lazy singleton | Model loaded once at startup; thread-safe for concurrent requests |

## Quick Start

### 1 — Install dependencies
```bash
pip install torch torchvision fastapi uvicorn python-multipart \
            opencv-python-headless Pillow numpy streamlit plotly
```

### 2 — Train the model

**Option A: Real PlantVillage dataset**
```bash
# Download from: https://www.kaggle.com/datasets/emmarex/plantdisease
# Extract so that folders like PlantVillage/Tomato___Early_blight/ exist

python scripts/train.py \
    --data_dir /path/to/PlantVillage \
    --backbone mobilenet_v3_small \
    --epochs 30

# EfficientNet-B0 for higher accuracy (slower):
python scripts/train.py \
    --data_dir /path/to/PlantVillage \
    --backbone efficientnet_b0 \
    --epochs 40 --lr 5e-4
```

**Option B: Synthetic data (demo / CI — no dataset needed)**
```bash
python scripts/train.py --synthetic --epochs 8
```

Expected results on real PlantVillage with pretrained ImageNet weights:
- MobileNetV3-Small: **~94% val accuracy** in 25 epochs
- EfficientNet-B0:   **~96% val accuracy** in 35 epochs

### 3 — Start FastAPI
```bash
uvicorn api.app:app --reload --port 8000
# Open: http://localhost:8000/docs
```

### 4 — Start Streamlit UI
```bash
streamlit run streamlit_app.py
# Open: http://localhost:8501
```

## API Reference

### `POST /analyze`
Analyze a base64-encoded image.

```python
import requests, base64

with open("tomato.jpg", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

resp = requests.post(
    "http://localhost:8000/analyze",
    json={"image_b64": b64, "crop_hint": "tomato"}
)
print(resp.json())
```

### `POST /analyze-upload`
Multipart file upload — ideal for mobile (React Native / Flutter).

```bash
curl -X POST http://localhost:8000/analyze-upload \
     -F "file=@tomato.jpg;type=image/jpeg" \
     -F "crop_hint=tomato"
```

### `GET /health`
```json
{
  "status": "ok",
  "model_loaded": true,
  "classes": 38,
  "inference_mode": "nn+heuristic",
  "device": "cpu"
}
```

### `GET /classes`
Returns the full list of disease class names the model was trained on.

## Freshness Signal Breakdown

| Signal | Weight | How it's measured | Interpretation |
|--------|--------|-------------------|----------------|
| `saturation` | 22% | HSV S channel mean | Vivid colour = fresh; grey/dull = aged |
| `greenness`  | 22% | LAB b* axis (negative = green) | Chlorophyll present = fresh |
| `browning`   | 20% | Brown pixel fraction | Enzymatic oxidation = decay |
| `sharpness`  | 18% | Laplacian variance | Turgid cells = crisp edges = fresh |
| `brightness` | 10% | HSV V channel mean | Optimal ~160/255; extremes = bad |
| `uniformity` | 8%  | Hue coefficient of variation | Spots/lesions break uniformity |

Weights configurable in `config.py → FRESHNESS_WEIGHTS`.

## Grading Thresholds

```python
# config.py
GRADE_THRESHOLDS = {
    "A": {"min_freshness": 68.0, "max_disease_prob": 0.25},
    "B": {"min_freshness": 38.0, "max_disease_prob": 0.70},
    # C = fallback
}
```

Shelf-life table: `SHELF_LIFE_TABLE` in `config.py`
Disease shelf penalties: `DISEASE_SHELF_PENALTIES` in `config.py`

## Extending for Production

| Goal | How |
|------|-----|
| Real PlantVillage data | Point `--data_dir` at the extracted dataset |
| GPU inference | `export DEVICE=cuda` before starting FastAPI |
| EfficientNet-B0 | `--backbone efficientnet_b0` in train.py |
| More disease classes | Use the full 54-class PlantVillage; adjust nothing else |
| Custom freshness model | Replace `FreshnessEstimator` with an ONNX/TFLite regression model |
| Calibrate thresholds | Edit `GRADE_THRESHOLDS` and `FRESHNESS_WEIGHTS` in `config.py` |
| Mount into existing FastAPI | `from api.app import app as cq_app; main_app.mount("/cq", cq_app)` |
